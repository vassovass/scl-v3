## Guidelines

1. **All artifacts must have**:
   - Context header at top (date, status, scope)
   - Changelog table at bottom
   - Descriptive filename

2. **When to create artifacts**:
   - Complex decisions requiring research
   - Multi-step implementations
   - Business/market analysis
   - Feature planning

3. **Version control**:
   - All artifacts are committed to git
   - Major changes logged in changelog table
